## 회사 개요
- **회사명:** Cycloid  
- **설립연도:** 2016년  
- **본사 위치:** 프랑스(렝스·파리) 및 완전 원격(유럽 전역)  
- **산업 분야:** 인프라스트럭처 소프트웨어, DevOps 플랫폼  
- **주요 제품:**  
  - Cycloid Platform (IaC 자동화 엔진, CD/CI 파이프라인, 거버넌스, 비용 최적화)  
  - Terraform 및 Kubernetes 통합 모듈  
- **팀 규모:** 약 50~100명(개발·운영·영업·고객지원 혼합)  

## 미션과 가치
- **미션:**  
  *“디지털 전환을 가속화하기 위해 복잡한 클라우드 인프라 관리를 단순화하고 자동화한다.”*  
- **핵심 가치:**  
  - **오픈 소스 기여:** 커뮤니티 중심 개발, 투명한 코드 공유  
  - **협업:** 다학제 팀 간의 긴밀한 협업과 지식 공유  
  - **고객 중심:** 사용자 피드백에 기반한 지속적 개선  
  - **자동화 우선:** 수작업 최소화, 반복 가능한 워크플로우 구축  
  - **품질 및 안정성:** 테스트·모니터링으로 높은 가용성 보장  

## 최근 뉴스 또는 변화
- 2022년 시리즈 A 펀딩(약 6백만 달러) 성공으로 R&D 및 글로벌 확장 가속  
- GitLab, HashiCorp Terraform과의 공식 파트너십 체결  
- 2024년 Cycloid 2.0 출시:  
  - 거버넌스·컴플라이언스 기능 강화  
  - 드리프트(drift) 탐지·알림 모듈 추가  
  - 멀티 클라우드 비용 최적화 대시보드 제공  

## 역할 맥락과 제품 관여도
- **팀 구조:**  
  - Core Platform 팀(5~8명 규모)의 Go 백엔드 엔지니어  
  - DevOps·SRE, 프론트엔드, UX 디자이너, 고객 성공 매니저와 긴밀 협업  
- **주요 업무:**  
  - Go 기반 마이크로서비스 설계·개발·운영  
  - Terraform Provider 및 모듈 확장  
  - Kubernetes Operator/Controller 개발  
  - 내부 API(GRPC/REST) 및 GraphQL 서비스 구현  
  - 고가용성·수평 확장성 보장, 성능 최적화  
  - CI/CD 파이프라인(GitLab CI, GitHub Actions) 및 자동화 스크립트 유지보수  

## 예상 면접 주제
1. **Golang 전문 지식**  
   - 고루틴·채널 활용 패턴, 컨텍스트(`context.Context`) 관리  
   - 에러 핸들링(`errors`, `pkg/errors`, `fmt.Errorf`) 및 로깅  
   - 인터페이스 설계, 의존성 주입(DI) 기법  
   - 모듈화·패키지 구조 설계  
2. **분산 시스템 설계**  
   - 이벤트 기반 아키텍처, 메시지 큐(RabbitMQ/Kafka)  
   - 마이크로서비스 간 통신 전략(gRPC vs REST vs GraphQL)  
   - 장애 격리(﻿circuit breaker), 재시도 전략  
3. **인프라스트럭처 자동화(IaC)**  
   - Terraform 플러그인 SDK 경험  
   - Kubernetes Operator 패턴  
   - 클라우드 API(AWS/GCP/Azure) 연동  
4. **DevOps & SRE**  
   - CI/CD 파이프라인 설계 및 최적화  
   - 모니터링·알림(Prometheus, Grafana, ELK)  
   - 블루/그린·카나리아 배포 전략  
5. **코드 품질 및 테스트**  
   - 단위 테스트, 통합 테스트, 목(mock) 사용  
   - 코드 리뷰 베스트 프랙티스  
   - Static Analysis 도구(go vet, golangci-lint)
